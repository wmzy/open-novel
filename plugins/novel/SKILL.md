# 小说写作技能

你正在辅助创作一部中文长篇小说。本文件是写作质量的核心约束：它定义"什么是好"，并给出可操作的诊断方法。每一条技法都配有"差→好"对照范文——请照此校准你的输出。

---

## 一、核心写作原则

### 1. Show, Don't Tell（进阶版）

不要用形容词报告情绪或状态，而要用**可观察的动作、物件、感官细节**让读者自己得出结论。情绪不是被命名的，而是被泄露的。

> **差**：林晚很伤心。她想起了去世的母亲，眼泪止不住地流下来，心中充满了思念。
>
> **好**：林晚把那件洗得发白的棉袄叠成三折，又打开。樟脑丸的气味漫出来，她按住胸口，指节陷进布料，像是怕里面的什么东西也跟着散了。

**要点**：伤心从未被说出。读者通过"叠了又打开"的反复动作、樟脑丸的嗅觉、按住胸口的细节，自己读出丧母的痛。报告"思念"是廉价；让读者思念才是技艺。

### 2. 感官沉浸

每个场景至少调动两种以上感官。视觉最廉价，嗅觉、触觉、听觉往往更难忘。感官细节要**具体到能反驳**——不是"香"，是"蒜末呛锅的香"；不是"冷"，是"石板从脚心往上蹿的冷"。

> **差**：厨房里很热闹，有各种各样的声音和气味，让人感到温馨。
>
> **好**：油在铁锅里爆开，溅到灶台上嘶嘶作响。她手腕一抖，蒜末落进去，呛人的香气立刻漫上来，盖过了窗台上那盆已经发蔫的葱。灶火映着她的脸，汗顺着鬓角滑下来。

### 3. 对话技艺

好对话的精髓在**潜台词**——角色真正想说的，从不在台词里。靠动作、停顿、回避来承载言外之意。避免"问答式"直给，避免角色把心理活动直接说出来。

> **差**：
> "你到底爱不爱我？"她焦急地问。
> "我当然爱你，我发誓！"他坚定地回答。
>
> **好**：
> "明天的事，你别管了。"他把钥匙推到桌中间。
> 她没接。"晚饭呢？"
> "随便。"
> 钥匙搁在那儿，一直搁到他出门。门关上的瞬间，她才伸手去够，又缩了回来。

**要点**：没有一个"爱"字，但留恋、试探、决裂全在。钥匙是她一直没敢碰的那段关系。

### 4. 节奏与句法

句长控制节奏：**长句铺陈、慢镜头；短句、断句制造冲击与速度**。动作戏用动词链短句，抒情段落允许长句延展。注意——删除一切不承载信息的副词（"快速地""忽然地""慢慢地"），让动词本身携带速度。

> **差**：他快速地奔跑过去，但是路面十分湿滑，所以他不慎摔倒在地，然后他艰难地爬了起来，继续向前跑去。
>
> **好**：他跑。石板湿，鞋底一滑——膝盖砸下去，疼得发白。他爬起来。接着跑。

## 二、反 AI 味诊断（模式识别）

LLM 的通病不是某个词，而是**六种写作模式**。成稿前请逐条自检，命中任一条即重写。

### 模式 1：抽象情绪标签
用"愤怒/悲伤/绝望/震惊"等情绪名词直接报告，替代了本应由细节唤起的感受。

> **差**：听到这个消息，她感到无比的震惊和悲痛，一时间无法接受这个残酷的现实。
>
> **好**：她手里的电话听筒没挂。嘟——嘟——嘟——，忙音一声接一声。她盯着墙上的钟，秒针走了整整一圈，她才把听筒放回去，放得很轻，像怕惊醒什么。

### 模式 2：模板化心理独白
"他不禁想……""她陷入了深深的回忆""命运为何如此不公"——这是 AI 最爱套的内心戏模板，空洞且公式化。真正的记忆是**碎片化、不完整、被某个具体物件勾起**的。

> **差**：望着窗外的雨，他不禁陷入了深深的回忆之中，往事如潮水般涌上心头，让他感慨万千。
>
> **好**：二十年了，那张脸他还是记不全。使劲想，只想起一截下颌，和领口上那颗发绿的铜扣。雨打在玻璃上，他没开灯。

### 模式 3：排比堆砌
用三四个结构相同的比喻连缀，制造"文采"的假象，实则稀释了每一个意象。**好描写往往只需要最准的那一个**。

> **差**：她的笑容如春风般温暖，如阳光般明媚，如花朵般绚烂，如星辰般璀璨，令人沉醉。
>
> **好**：她笑起来，右边那个酒窝比左边深一些。

### 模式 4：万能形容词
"美丽而宁静""独特而迷人""神秘而古老"——形容词没有锚点，可以套在任何对象上，等于没写。**每个形容词都要能被一个具体细节替换或支撑**。

> **差**：这是一个美丽而宁静的江南小镇，有着独特的历史韵味和迷人的风情。
>
> **好**：镇子只有一条街。青石板的缝里长着草，午后连狗都懒得叫。河埠头的石阶绿得发黑，年深日久，踩上去打滑。

### 模式 5：转折连词滥用
"然而……但是……不过……尽管如此"——AI 习惯用转折词硬接逻辑。好的叙述靠**意象的并置**自然产生张力，少用连词。

> **差**：然而他并没有放弃，但是他的身体已经十分疲惫，不过他还是咬紧牙关坚持了下来。
>
> **好**：他没停下。腿早不听使唤了，膝盖每弯一下都像灌了铅。

### 模式 6：情节概括代替演绎
"经过一番激烈的搏斗，他终于战胜了敌人"——这是大纲语言，不是小说语言。**读者要的是过程，是一个个具体的回合**。

> **差**：经过一番激烈的较量，他终于凭借着顽强的意志打败了对手，取得了胜利。
>
> **好**：第三拳落空，他借着惯性撞进对方怀里，肘尖死死顶住喉结。听见一声含混的闷响，那人膝盖一软，慢慢矮下去。他没松手，直到确认对方不再挣。

## 三、场景写作技法

### 1. 动作场景
用**短句、动词链、身体感受**构建速度感。一个动作配一个后果（声音/疼痛/位移）。不要解说招式的意义，让动作本身说话。

> 路灯下只有两个人的影子。他先动的——侧身，肩膀擦着对方的拳风过去，反手扣住腕子往下一压。骨头在掌心咯吱响。对方闷哼一声，另一只手已经摸到了腰后。他没等那把刀出来，膝盖顶上去，听见牙关磕碎的声音。

### 2. 情感场景
情感的高潮要**克制**。越是剧烈的情绪，越要写微小的动作与物件，让情绪在缝隙里炸开。避免嚎啕、嘶吼、声泪俱下的直接描写。

> 她在抽屉最里面摸到一个铁盒。打开，是一缕头发，用红绳系着，已经发黄。她坐到地上，把铁盒搁在膝盖上，很久没动。窗外有人在喊孩子回家吃饭。她抬起手，才发现脸早就湿了。

### 3. 转场
转场靠**一个承上启下的意象**完成，不要用"三天后""与此同时"这类生硬标记。让一个感官或物件把两个时空缝起来。

> （上段结尾）他把那张照片塞回信封，封口压了又压。
>
> （下段开头）信封还是摊在桌上。十年了，纸都脆了，边角卷起来，像一片要落的叶子。门铃响的时候，他正用镇纸压着它。

### 4. 环境场景
环境不是背景板，而是**情绪的放大器**与**角色的处境**。每个环境细节都应服务于氛围或主题，避免百科式罗列。

> 巷子窄得只能过一个人。两侧的山墙长满青苔，头顶的电线绞成一团，漏下几缕灰白的天光。地上湿，是那种渗了很久、分不清是雨还是地下水的湿。他贴着墙走，砖缝里的凉气透过衬衫，一直爬到后颈。

## 四、阶段工作流与质量检查

创作分六个阶段。每个阶段的"保存文件、推进 stage"动作由系统指令完成；**你的职责是在进入下一阶段前，逐条核对质量清单**。未达标则留在当前阶段补足。

### 阶段 1 · 概念 (concept)
产出 `.novel/concept.md`。须包含：一句话梗概、五句扩展、核心冲突、道德前提、主角的两难抉择。

**质量清单**：
- [ ] 核心冲突是**具体的人际或处境矛盾**，而非抽象主题（"人性"不是冲突，"母亲要他继承药铺、他想进城学画"才是）
- [ ] 两难抉择**两头都有代价**，没有显然正确的答案
- [ ] 能用一句话讲清"谁，想要什么，被什么阻挡，会付出什么代价"

### 阶段 2 · 世界观 (world)
产出 `.novel/world-building.md`。

**质量清单**：
- [ ] 有**明确的生活细节**（货币、饮食、出行、婚丧），而非只有宏大设定
- [ ] **权力结构与经济逻辑**清晰：谁掌权、谁被压迫、人们靠什么活——这些设定能**自然生出冲突**，而非只是背景板
- [ ] 交通/通讯/技术有**明确边界**：这些限制本身就能成为情节张力（迟滞、隔绝、信息差）
- [ ] 力量/规则体系有**代价与限制**，不是万能
- [ ] 世界与主题呼应（一个讲"束缚"的故事，世界本身就应充满规矩与藩篱）

### 阶段 3 · 角色 (characters)
产出 `.novel/characters/profiles.md`。

**质量清单**：
- [ ] 每个主要角色有**驱动三角**：外在目标 / 内在弱点 / 转变弧线
- [ ] 反派动机**合理可理解**，不是纯粹的恶；他与主角的冲突根植于价值观差异
- [ ] 角色有**辨识度**：删掉名字，光凭说话方式与习惯动作能认出是谁

### 阶段 4 · 大纲 (outline)
产出 `.novel/outline.md`（含简要三幕 + 分章规划）。

**质量清单**：
- [ ] 结构有**因果链**：每个重大转折都是前文事件引发的，不是巧合
- [ ] 章节分布与目标字数匹配（目标字数 ÷ 章节数 ≈ 每章字数）
- [ ] 有至少一处**伏笔**与一处**回扣**，记录到 `foreshadow.json`

### 阶段 5 · 场景 (scenes)
产出 `.novel/scenes.md`。

**质量清单**：
- [ ] 每个场景有**明确的视点角色**与**该场景的目标**
- [ ] 主动场景（目标→冲突→灾难/转折）与被动场景（反应→困境→新决定）**交替**，避免连续同型
- [ ] 每个场景能回答："这场戏如果不写，读者会漏掉什么？"

### 阶段 6 · 写作 (writing)
逐章产出 `.novel/chapters/第N章.md`。**每章完成后必做**：写 `第N章.summary.md`（≤200 字压缩摘要）、更新 `state.json`（角色位置/情绪/新知/关系/时间线）。

**质量清单**：
- [ ] 通过"反 AI 味六模式"自检（见第二节），命中即改
- [ ] 每个场景至少调动两种感官，对话有潜台词
- [ ] 角色行为与 `state.json` 中记录的状态、已知信息**一致**（不能知道还没发生的事）
- [ ] 伏笔的埋设/回收与 `foreshadow.json` 同步，无遗漏或矛盾
- [ ] 章末留有**钩子**（悬念/未决问题），牵引下一章

## 五、文件结构说明

所有内容存放于项目目录的 `.novel/` 下：

| 文件 | 用途 |
|---|---|
| `concept.md` | 故事概念（梗概、冲突、道德前提、两难） |
| `world-building.md` | 世界观（地理、社会、规则、文化） |
| `characters/profiles.md` | 角色档案（驱动三角、弧线、关系） |
| `outline.md` | 大纲（三幕结构 + 分章规划） |
| `scenes.md` | 场景设计（视点、目标、节拍） |
| `chapters/第N章.md` | 第 N 章正文 |
| `chapters/第N章.summary.md` | **第 N 章压缩摘要**（≤200 字，每章必写，供后续章节保持连贯） |
| `state.json` | **全局状态**：各角色的位置/情绪/已知信息/关系、时间线、活跃伏笔、`lastUpdatedChapter`、`updatedAt`（每章后更新） |
| `foreshadow.json` | 伏笔追踪（id/内容/状态 pending\|planted\|resolved/埋设/回收章节） |
| `summary.md` | 全书摘要（已完成章节、进度、重要转折） |
| `config.json` | 项目配置 |

### state.json 结构（关键字段）
```json
{
  "characters": [
    { "name": "林晚", "location": "老宅", "emotion": "压抑",
      "knows": ["母亲遗物在铁盒里"], "relationships": { "陈砚": "疏离" },
      "lastAppearance": 3 }
  ],
  "timeline": "第3章末·黄昏",
  "activeForeshadows": [1],
  "lastUpdatedChapter": 3,
  "updatedAt": "2026-07-02T10:00:00Z"
}
```

### 第N章.summary.md 用途
系统在写作阶段会把**前若干章的压缩摘要**注入上下文，让 agent 在不重读全文的情况下保持人物状态、情节与伏笔的连贯。因此摘要要**只压缩事实**（谁做了什么、得知了什么、状态如何变化），不含抒情与铺陈。
