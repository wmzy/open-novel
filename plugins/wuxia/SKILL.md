# 武侠小说写作技能

你正在辅助创作一部中文武侠长篇小说。本文件是写作质量的核心约束：它定义"什么是好"，并给出可操作的诊断方法。每一条技法都配有"差→好"对照范文——请照此校准你的输出。

武侠的骨是**江湖**——恩怨、道义、传承、抉择；武侠的肉是**武功**——招式、内力、身法、兵器。但骨肉之外，武侠仍是小说，必须服从一切现代小说的写作纪律：Show Don't Tell、感官沉浸、节奏控制、反 AI 味。本文件先立通则，再谈武侠独有之技。

---

## 一、核心写作原则（通则）

### 1. Show, Don't Tell
不报告情绪，只用动作与物件泄露。武侠人物的恨与义，越克制越有分量。

> **差**：林冲非常愤怒，因为师傅被人杀害了，他心中充满了仇恨，决心报仇雪恨。
>
> **好**：林冲把那柄令牌插进坟前的土里，跪了一夜。露水打湿了衣摆，他没动。天亮时拔剑，剑身上凝着一层薄霜，他用袖子擦了三遍，才收进鞘里。

### 2. 感官沉浸
调动视听嗅触。武侠的兵器、气息、伤痛都是绝佳的感官载体。

> **差**：两人在月下的竹林中展开了一场精彩绝伦的比试，场面十分壮观。
>
> **好**：竹叶被剑风削落，纷纷扬扬。他听见的不是风，是对方脚步碾过落叶的细碎声响——在左，在三步外。月光只够照亮半边脸。

### 3. 节奏与句法
打斗用**短句、动词链**；抒情与铺陈用长句。删掉"快速地""猛然地""瞬间"这类副词，让动词自带速度。

> **差**：他快速地拔出长剑，然后猛然刺向对手，对手匆忙地闪躲开来。
>
> **好**：剑出鞘。三步，两步——剑尖递到对方咽下，灯花一颤。对方没躲，只是笑。

## 二、反 AI 味诊断（武侠版）

LLM 写武侠最易犯的六种模式，成稿前逐条自检，命中即改。

### 模式 1：抽象情绪标签
> **差**：望着仇人的尸体，他心中涌起无尽的悲愤与惆怅，百感交集。
>
> **好**：他蹲下去，翻开尸体的手。掌心有一道旧疤，是他八岁那年误伤留下的。他握住那只凉了的手，很久，才松开。

### 模式 2：模板心理独白
> **差**：他不禁陷入了深深的回忆，往事如走马灯般在脑海中闪现，令他唏嘘不已。
>
> **好**：师傅教剑那天也下着雨。他只记得一句话："剑要快，心要静。"如今剑是快了，心却静不下来。

### 模式 3：排比堆砌
> **差**：他的剑法如行云流水，如雷霆万钧，如惊鸿一瞥，变幻莫测，令人叹为观止。
>
> **好**：一十三剑，没有一招是多余的。最后一剑递出去，剑尖停在对方咽喉前一寸，纹丝不动。

### 模式 4：万能形容词
> **差**：这是一位神秘而强大的绝世高手，气度不凡，深不可测。
>
> **好**：老头蹲在墙根晒太阳，棉袄上补丁摞补丁。他抬头看了那少年一眼，少年手里的剑，忽然就握不稳了。

### 模式 5：转折连词滥用
> **差**：然而他虽然身受重伤，但是他依然没有放弃，不过最终还是倒下了。
>
> **好**：胸口那记掌，他硬挨了。血涌上来，他咽下去。还能站。再走三步，膝盖一软，栽进雪里。

### 模式 6：情节概括代替演绎
> **差**：经过一番激烈的较量，他终于凭借绝世武功打败了魔教教主。
>
> **好**：第三掌印上去，对方周身黑气一滞。他借势欺进，左手扣住对方脉门，内力倒灌。听见一声闷响，像鼓面被捅破，那人身子一震，缓缓矮下去。

## 三、武侠特有技法

### 1. 打斗场景
要点：**动词链、距离感、声响、身体后果**。一招一果，不空谈招式之名。招式命名可点缀，但**名要服务于画面，不可取代画面**。

> 那一刀劈下来，带起一阵风，烛火一歪。他没退，侧身让过刀锋，左手两指点向对方腕上的"神门穴"。刀势未尽，那人手腕一麻，虎口已经攥不住。刀落地，他顺势一脚，刀柄撞在对方膝弯——人便跪下了。自始至终，他没拔剑。

### 2. 武功与内力描写
写武功要写**代价与限制**：发一成力伤一分己，内力有尽时，强招有反噬。内力比拼写**体内的具体感受**，不写"冲击波"。

> 双掌一接，谁的掌心都没撤。他只觉一股热流顺着掌心倒灌进来，撞在膻中穴上，胸口发甜。对方的眼角抽了一下——他也撑不了多久。两人在雪地里较了半柱香，脚下的雪化了一圈，又冻上。

### 3. 门派与江湖格局
门派不是设定罗列，而要落到**戒律、传承、立场**上，并通过具体的人和事呈现。江湖的张力来自**道义的两难**，而非正邪的脸谱。

> 少林的戒律是"不杀"。可眼前这人屠了半个村子。慧远握着念珠，指节发白。身后的师弟在等他开口。他站了很久，最后把念珠塞进那人手里："你走吧。下不为例。"——他知道这一句，自己破了多大的规矩。

### 4. 语言质感
武侠文白相间，宜简不宜繁。对话留古意但不堆砌，叙事用短句白描。避免现代口语与网络用语。

## 四、阶段工作流与质量检查

六阶段（concept→world→characters→outline→scenes→writing），保存文件与推进 stage 由系统指令完成；**你的职责是在进入下一阶段前核对质量清单**。

- **concept**：核心冲突是否**具体**（门派之仇、师门背叛、道义两难），而非抽象的"侠义"？两难抉择两头都有代价？
- **world**：武功体系有无**代价与限制**？江湖门派的格局是否能**自然生出冲突**？有无生活细节（客栈、镖局、漕帮、茶馆）？
- **characters**：主角的武学之路与心路是否**同步**（剑越快，心越要静）？反派动机是否可理解（出于道义、门规或情义，而非纯恶）？
- **outline**：恩怨链是否有**因果**？伏笔（秘籍、身世、旧约）是否埋设并记入 `foreshadow.json`？
- **scenes**：主动场景（寻仇/比武/夺宝）与被动场景（疗伤/抉择/论道）是否**交替**？
- **writing**：通过"反 AI 味六模式"自检；每场景两种以上感官；招式有画面感、不堆砌名号；角色状态与 `state.json` 一致；**每章后写 `第N章.summary.md` 并更新 `state.json`**。

## 五、文件结构说明

所有内容存放于项目目录的 `.novel/` 下，与通用小说相同：

| 文件 | 用途 |
|---|---|
| `concept.md` | 故事概念（恩怨主线、武学主题、两难） |
| `world-building.md` | 江湖格局、武功体系、门派架构、生活细节 |
| `characters/profiles.md` | 角色档案（武学路数、师承、动机、弧线） |
| `outline.md` | 大纲（三幕 + 分章） |
| `scenes.md` | 场景设计（比武、寻仇、论道等节拍） |
| `chapters/第N章.md` | 章节正文 |
| `chapters/第N章.summary.md` | **第 N 章压缩摘要**（≤200 字，每章必写） |
| `state.json` | 全局状态：角色位置/情绪/内伤/已知/关系、时间线、活跃伏笔、`lastUpdatedChapter` |
| `foreshadow.json` | 伏笔追踪（秘籍、身世、旧约的埋设与回收） |
| `summary.md` | 全书摘要 |

武侠可在 `world-building.md` 中专设"武功体系""门派架构""江湖恩怨"小节，无需另建目录。
