# 设计：写作定向上下文注入与角色体系增强

**日期**：2026-07-04
**范围**：open-novel 工具完善（P0 写作定向上下文 + 角色体系增强）
**动机**：写作阶段 prompt 不注入角色档案，agent 看不到"本章出场角色是谁、他们是谁、这章发生什么"——用户精心造的人物冰山无法抵达水面，导致扁平化、工具人化。

---

## 1. 问题诊断（代码证据）

### 1.1 实际生效的 prompt 组装器

`src/agent/prompt-composer.ts` 是唯一活代码——`runs.ts:282`、`rewrite.ts:69` 均 import 它。`src/prompts/compose.ts` 是死代码（零调用点），但误导了 `docs/工具使用指南.md`。

### 1.2 写第 N 章时实际注入的四层（`buildWritingContextLayers`，prompt-composer.ts:356）

| 层 | 内容 | 示例项目实际状态 |
|---|---|---|
| 核心设定 | `concept.md` + `world-building.md`（world 截断到 16KB） | ✓ 有 |
| 状态 | `state.json`（角色位置/情绪/已知/关系） | ✗ 空（`{characters:[]}`） |
| 滚动摘要 | 最近 3 章详摘 + 更早简摘 | ✗ 未开写 |
| 活跃伏笔 | `foreshadow.json` | ✗ 空（`{items:[]}`） |

### 1.3 写第 N 章时看不到的内容（核心缺口）

| 该知道的 | 注入？ | 证据 |
|---|---|---|
| 第 N 章大纲片段 | ✗ | `outline-detailed.md` 仅在 `Project Files` 列文件名，内容不注入 |
| 第 N 章场景节拍 | ✗ | `scenes.md` 同上 |
| 本章出场角色是谁 | ✗ | 无识别机制 |
| 出场角色档案（性格/动机/声口/小动作） | ✗ | `characters/profiles/*.md` 多份，只列文件名，零注入 |

`STAGE_INSTRUCTIONS['writing']`（prompt-composer.ts:60）通篇规定"写完后的维护"（摘要/state/伏笔），**无一句要求"写前读大纲/场景/角色档案"**。这是扁平化的技术根源。

---

## 2. 设计概览

在写作阶段新增两层上下文 + 一项可选资产，并清理死代码与文档：

```
核心设定 → 状态 → ★本章大纲块（新）★ → ★本章出场角色层（新）★ → 滚动摘要 → 活跃伏笔
```

P0 闭环：写第 N 章时自动注入——本章大纲块 + 出场角色档案（分级）+（若有）声口样本。

---

## 3. 详细设计

### 3.1 出场角色识别（三级回退）

写第 N 章时，确定"谁在场"的三级机制，逐级降级保证总有结果：

1. **outline-meta.json**：`chapters[N].pov` 字段（最准，POV 角色）
2. **大纲块表格**：解析 `outline-detailed.md` 第 N 章块的"出场角色 / POV"行
3. **角色名匹配**：大纲块全文做名字匹配（复用现有 `readCharacterNames`，返回 profiles 索引的若干名字）

**分级**（决定注入体积）：
- **POV 角色**（来自第 1 或 2 级）→ L1 完整注入
- **主要互动者**（大纲块显式列出的具名角色）→ L1 完整注入
- **其余**（第 3 级名字匹配扫到、但不在显式列表）→ L2 速查

**边界**：
- 连读章节（`第16-17章`、`第27-30章`）解析为章号范围，取角色并集
- 章号超出大纲范围：不注入大纲块，提示 `[第N章未在 outline-detailed.md 中规划]`
- 三级全失败：退化为注入 `state.json` 中 `lastAppearance` 接近 N 的角色

### 3.2 分级注入与 token 预算（方案 B）

| 等级 | 触发条件 | 注入内容 | 单角色预算 |
|---|---|---|---|
| **L1 完整** | POV 角色；大纲块显式具名出场角色 | profile 按标题分块提取关键段 | ≤6KB（超长截断） |
| **L2 速查** | 第 3 级名字匹配扫到、未在显式列表的角色 | 名字 / 定位 / 核心驱动力 / 标志细节 | ~150 字 |

**L1 关键段优先级**（按"由人物演绎故事"需求排序，高→低）：

1. **出身与经历**（侠本是百姓——不是"侠客折射平民"，而是侠的平民本质本身。这一段是人物社会属性的根）
2. **驱动力三角**（外在目标 / 内在需求 / 核心缺陷——决定行为逻辑）
3. **性格 + 语言行为习惯**（声口与小动作——防扁平）
4. 基本信息、外貌（次优先）

**截断跳过**：成长弧线、武学详解、与各角色关系详表（对单章写作非必需，agent 需要时自行 Read）。

**角色层总预算**：软上限 **20KB**（约 10K token）。多角色章节按 POV → 主要互动者 → 次要顺序填入，超出则后续降级为 L2。硬约束：宁可次要角色变速查，也不让总上下文破 50KB 触发已知 token 退化。

**降级实现**：
- profile 已用 `## 标题` 组织，用 markdown 标题切片器提取关键段
- 标题结构不清晰（如群像文件 `帮会代表人物.md` 含多角色）：退化为"前 2KB + `[完整档案见 …]`"

### 3.3 大纲块提取

从 `outline-detailed.md` 解析 `#### 第N章：标题` 锚点，截取到下一个 `####` 或 `###` 之前。

**边界**：
- 连读章节 `第16-17章` / `第27-30章`：章号范围匹配，写范围内任意章号都命中该块
- 多 POV 合并章节：按当前章号在范围内的位置提示"本章 POV 倾向"
- 章号超出大纲范围：不注入，提示 `[第N章未在 outline-detailed.md 中规划]`
- 大纲文件不存在：退化为 `outline-brief.md` 对应卷的概述

**实现顺序**：提取大纲块 → 从块中识别出场角色（§3.1）→ 分级注入角色档案 + 大纲块本身。

**注入位置**：在出场角色层之前（先有"这章发生什么"，再有"谁在场"）。

**注入格式**：
```
### 本章大纲（第N章）
{大纲块原文}

> 严格按大纲推进。若需偏离（增删事件、调整节奏），在回复里说明原因。
```

"偏离需说明"防止 agent 擅自改情节，同时为"由人物演绎故事时发现大纲需调整"留弹性。

### 3.4 声口库机制（可选资产）

防"众口一词"——AI 写多个角色若无声音样本，可能一个腔调。

**存储**：`characters/voices/{角色名}.md`，每人 3-5 段样本：
- 独白（内心戏的声音）
- 对话（对外说话的声音）
- 极端情境（愤怒/悲痛/恐惧——AI 最易写成通腔的时刻）

每段 100-200 字，标注情境。

**注入**：L1 角色若有 voices 文件，作为 profile 附录注入（计入该角色 6KB 预算，不另开）。L2 角色不注入声口。

**可选性**：声口库不强制存在。`STAGE_INSTRUCTIONS['characters']` 增提示"为核心角色建立 voices/{角色名}.md"。未建文件的角色只注入 profile。示例项目现无此资产，不影响 P0 跑通；补建后自动生效。

### 3.5 清理死代码 + docs 对齐

- **删除** `src/prompts/compose.ts`（零调用点）
- **更新** `docs/工具使用指南.md` 第 4.2 节：四层 → 六层（加大纲块、出场角色层），注明角色档案按分级注入
- **更新** 批量写作示例：移除"用户每章手打'先读 outline、scenes、state'"——P0 后自动注入，用户只需说"写第 N 章"

### 3.6 writing 阶段指令修订

`STAGE_INSTRUCTIONS['writing']` 头部增"写章前"指引：

> **写章前**：本章大纲与出场角色档案已注入上方上下文。无需再 Read 这些文件——直接基于注入内容写作。仅在需要查阅未注入细节（如某角色完整弧线、某武学体系全貌）时才 Read。

解决两个问题：① 告知 agent 档案已在手，不必重复读；② 明确"什么该 Read、什么不该"，减少 token 浪费。

## 4. 非目标

- **不新增"愚公/平民建档模板"**：经讨论，模板化小人物日常会沦为工具人。侠不是独立于百姓的精英阶层——侠就是百姓的代表、普通人心中的火。因此不存在"侠客折射平民"的设计需求，角色档案的"出身与经历"段自然承载每个人的社会属性，无需另设平民模板。
- **不实现情节张力诊断 / 悬念地图 / 情节变体探索**：本轮聚焦 P0 闭环，这些留待后续。
- **不强制 state.json / foreshadow.json 填充**：注入层在文件为空时优雅降级（跳过该层），不阻塞写作。

---

## 5. 验收标准

1. 写第 N 章时，prompt 包含第 N 章大纲块原文（连读章节按范围匹配）
2. prompt 包含 POV 角色及显式出场角色的 L1 profile 关键段
3. 角色层总体积不超过 20KB 软上限
4. 三级回退全部失效时，降级为 state.json 最近出场角色，不报错
5. `src/prompts/compose.ts` 已删除，无残留引用
6. `docs/工具使用指南.md` 描述的六层与实际实现一致
7. 声口库存在时被注入，不存在时不阻塞

---

## 6. 风险与取舍

| 风险 | 缓解 |
|---|---|
| profile 标题结构不统一，分段提取可能失败 | 降级为"前 2KB + 指路"，保证有内容 |
| token 预算过紧，多角色章节信息不足 | POV 永远拿完整 L1；次要角色降级而非截断 POV |
| outline-meta.json 缺失（示例项目现状） | 三级回退的第 2、3 级不依赖它，仍可识别角色 |
| 声口库增加用户负担 | 可选资产，不强制；自动注入仅在文件存在时触发 |
