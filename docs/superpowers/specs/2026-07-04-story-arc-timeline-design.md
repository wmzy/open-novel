# 设计：故事脉络时间线视图（timeline + sequenceDiagram）

**日期**：2026-07-04
**范围**：open-novel 前端新视图「故事脉络」
**动机**：作者写长篇时难以宏观把握"全书怎么推进、各章谁与谁发生了什么"。需要一个可视化脉络——主图展示整体时间线，点开章节看角色交互细节。mermaid 已集成（`MermaidDiagram.tsx` + mermaid 11.16），章回/角色数据已结构化（`chapter-context.ts`），但缺一个把大纲组织成"脉络图"的层。

---

## 1. 设计概览

组合两种 mermaid 图，各司其职：

- **主图 timeline**：以卷为 section、章为节点，展示全书脉络。数据来自大纲现成字段（POV / 核心事件 / 出场角色），**纯函数生成，零 AI**。
- **章节子图 sequenceDiagram**：展开某章时显示该章角色交互。数据来自大纲每章新增的「角色交互」结构化字段，**AI 批量预填 + 视图内可修正**。

```
故事脉络视图
├─ [AI 批量生成交互] 按钮 + 进度（第 X/N 章）
├─ CollapsibleDiagram(timeline)        ← 全书脉络
└─ 章节折叠列表
   └─ 展开某章
      ├─ CollapsibleDiagram(sequenceDiagram)   ← 该章角色交互
      └─ [编辑] 按钮 → 文本框 → 保存写回大纲
```

入口：Sidebar 加「故事脉络」tab，与「写作/对话」平级。

---

## 2. 数据格式

### 2.1 大纲新增「角色交互」表格行

在每章现有表格（`POV` / `核心事件` / `出场角色` / `情感` / `写作提示`）新增一行 `角色交互`：

```markdown
#### 第3章：关卡
| 项目 | 内容 |
|------|------|
| POV | 武松 |
| 核心事件 | 在客栈因无通行凭证被守卫盘问，何九叔出手相助 |
| 出场角色 | 武松、何九叔 |
| 角色交互 | 武松→何九叔[冲突]：被盘问 · 何九叔→武松[善意]：出手相助 |
| 情感 | 尴尬、警觉 |
```

### 2.2 交互字段语法

- 多个交互用 ` · `（中点 + 两侧空格）分隔
- 单条格式：`主动方→被动方[类型]：动作描述`
- 类型枚举（9 种）：`冲突` / `合作` / `对话` / `试探` / `对决` / `善意` / `背叛` / `重逢` / `离别`
- 独角戏章节：字段值为 `（无）` 或留空

### 2.3 选型理由

选紧凑内联格式而非独立子表的理由：
- 不破坏现有大纲表格结构（一行内嵌）
- 正则即可解析，无需复杂 AST
- 人可读，便于手工修正
- AI 生成时只需填一格，prompt 简单

## 3. 数据流（三种生成器 + 一种修正）

### 3.1 timeline 源码（纯函数，无 AI）

**输入**：`outline-detailed.md` 全文
**解析**：
- `## 卷X` / `## 第X篇` → mermaid `timeline` 的 `section`
- `#### 第N章：标题` → section 下的节点
- 每章节点标注：章号 + 标题 + POV + 关键角色（来自表格「出场角色」行）

**输出示例**：
```
timeline
    title 《示例集》故事脉络
    section 第一卷
        第1章 启程前夜 : POV 武松 : 独角戏
        第3章 关卡 : POV 武松 : 武松、何九叔
    section 第二卷 ...
        第16章 论剑 : POV 武松 : 武松、西门庆
```

**复用**：`chapter-context.ts` 的 `extractChapterOutline` 章节锚点解析（含连读章节 `第16-17章` 范围处理）。

### 3.2 sequenceDiagram 源码（纯函数，无 AI）

**输入**：某章「角色交互」字段文本
**解析**：
- 按 ` · ` 切分多个交互
- 每条正则：`^(.+?)→(.+?)\[(.+?)\]：(.+)$`
- participant 去重（同一角色多次出现只声明一次）

**输出示例**（第3章）：
```
sequenceDiagram
    participant 武松
    participant 何九叔
    武松->>何九叔: 被盘问
    Note over 武松,何九叔: 冲突
    何九叔->>武松: 出手相助
    Note over 何九叔,武松: 善意
```

**降级**：
- 字段为空 / `（无）` / 无法正则解析 → 返回 `null`，视图提示"无交互数据或格式异常，点编辑修正"

### 3.3 AI 批量预填（一次性，走 SSE）

**职责**：遍历所有章节块 → 每章 AI 读「核心事件 + 出场角色」，输出符合 §2.2 语法的「角色交互」字段 → in-place 写回大纲 md（表格内插入新行）。

**幂等**：已有「角色交互」行的章节跳过，不重填。

**SSE 协议**：复用 `useRun` / SSE 基建。逐章回传 `{type:'progress', chapter:N, total:M}`；某章失败回传 `{type:'error', chapter:N, message}` 但不阻断；末尾回传 `{type:'done', filled:[...], skipped:[...], failed:[...]}` 汇总。

**AI prompt 要点**：
- 给出该章的 POV / 核心事件 / 出场角色原文
- 要求严格按 `主动方→被动方[类型]：动作` 格式，类型必须取自 9 种枚举
- 独角戏返回 `（无）`
- 多交互用 ` · ` 分隔

### 3.4 视图内修正

**职责**：编辑框改文本 → PUT 接口 → in-place 替换大纲该章「角色交互」行。

**边界**：
- 该章表格原本无「角色交互」行 → 插入新行（在「出场角色」行之后）
- 该章已有 → 替换该行内容
- 文本格式校验在前端（按 §2.2 正则），不合法则禁用保存按钮 + 提示

## 4. 组件与文件

### 4.1 新增

| 文件 | 职责 |
|---|---|
| `src/agent/timeline-builder.ts` | 纯函数：大纲→timeline 源码；交互字段→sequenceDiagram 源码；交互字段解析/校验 |
| `src/agent/timeline-filler.ts` | AI 批量预填：遍历章节→AI 生成→写回大纲 |
| `src/web/views/StoryArcView.tsx` | 故事脉络视图 |
| `src/api/routes/timeline.ts` | `GET /:project/timeline`（返回 timeline 源码 + 各章交互字段原文，sequenceDiagram 源码由前端纯函数按需生成，便于修正后即时重渲染）、`POST .../fill`（SSE 批量预填）、`PUT .../interaction`（修正单章） |

### 4.2 复用

- `MermaidDiagram` / `CollapsibleDiagram`（渲染）
- `chapter-context.ts` 的 `extractChapterOutline`（章节锚点解析，含连读章节范围处理）
- `useRun` / SSE 基建（批量预填长任务）
- prompt-composer 的 AI 调用通道（批量预填用同一 agent 通道）

---

## 5. 错误处理

- 大纲表格残缺 → timeline 降级为可解析的章节，跳过坏章（不报错）
- 交互字段格式错（无法正则解析）→ sequenceDiagram 区域提示"格式异常，点编辑修正"
- AI 预填某章失败 → 跳过标记，其余继续；SSE 末尾汇总失败列表
- mermaid 渲染失败 → `MermaidDiagram` 现有降级提示复用
- 大纲文件不存在 → timeline 返回空源码，视图提示"无大纲"

## 6. 测试策略

**纯函数为重点**（timeline-builder.ts）：
- 章节锚点：单章 / 范围章（`第16-17章`）/ 无标题章
- timeline 源码：section 划分、节点标注、空大纲降级
- 交互字段解析：多交互切分、类型枚举校验、独角戏空值、格式错的容错
- sequenceDiagram 源码：participant 去重、箭头方向、Note 生成

**集成**（timeline-filler.ts）：
- mock AI 验证写回格式正确
- 幂等：已有字段不重填

**API**（timeline.ts）：
- 三个端点的 happy path + 错误分支

---

## 7. 非目标

- **不做角色关系网拓扑图**（静态 flowchart）：本轮聚焦时间线脉络 + 角色交互时序，关系网留待后续
- **不解析已写章节正文提取交互**：sequenceDiagram 数据统一来自大纲「角色交互」字段，不双轨
- **不实现交互类型的统计/筛选**：9 种类型仅用于 sequenceDiagram 的 Note 标注，不做维度筛选

---

## 8. 验收标准

1. Sidebar 出现「故事脉络」tab，点开显示全书 timeline 主图（卷为 section，章为节点，标注 POV/角色）
2. 展开某章显示该章 sequenceDiagram（若有「角色交互」字段）
3. 「AI 批量生成交互」按钮走 SSE 逐章预填，进度可见，失败章节不阻断
4. 预填幂等：已有「角色交互」行的章节跳过
5. sequenceDiagram 旁「编辑」按钮可改文本并写回大纲 md
6. 无「角色交互」字段的章展开后提示，不报错
7. 大纲表格残缺/字段格式错时优雅降级，不崩溃
8. 纯函数单测覆盖：章节锚点、timeline/sequenceDiagram 源码生成、交互字段解析

---

## 9. 风险与取舍

| 风险 | 缓解 |
|---|---|
| AI 预填的交互类型不准（如把"试探"标成"对话"） | 9 种枚举收敛歧义；视图内可修正；幂等保证只预填一次 |
| 批量预填耗时长（30+ 章） | SSE 逐章回传进度；失败章节跳过不阻断；幂等保证中断后可续 |
| 大纲表格结构不统一（有的章无「出场角色」行） | 解析器对缺失行降级为空，timeline 节点仅标 POV |
| 修正写回大纲时破坏表格格式 | in-place 替换严格按行匹配；写前备份语义（复用现有 snapshot 机制） |
| sequenceDiagram 角色过多（群像章）拥挤 | mermaid 自动布局；超 6 角色时视图提示"角色较多，建议精简" |
